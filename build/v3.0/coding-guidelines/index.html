Coding guidelines

The design system has been built using Sass, a CSS pre-processor that adds program logic, variables and nesting to simplify long and complicated stylesheet files.

CSS Rules

To make style code easier to maintain, we use a few rules to keep it consistent.

Styles are defined one property-per-line, all lines end in a semi-colon, indented by 2 spaces, ordered alphabetically.

a {
  color #036;
  text-decoration: none;
}

A new element should always have an empty line separating it from the element above, unless stacking elements.

Never use IDs (#) for style, always classes (.) or attributes ([]).

Named elements should be followed by their related pseudo-classes, and then anonymous classes.

a,
[data-link] {
  color #036;
  text-decoration: none;
}

a:hover {
  text-decoration: underline;
}

.error {
  color: #d33;
}

For numbers less than one, do not use a leading 0.

.disabled {
  opacity: .6;
}

Sass Rules

We use the SCSS (CSS-compatible) syntax of Sass.

Properties should be ordered:

  1. @extend
  2. @include
  3. Own properties
  4. Child elements

Where possible, follow the "inception rule": no nesting more than 3 levels deep.

Always use $variables in place of raw color values, and remember that these can still be used in lighten() and darken() SASS functions, as well as rgba() values.

$navy: #036;
$highlight: #69e;

h1,
p {
  @extend %wrapper;
  @include rem(padding-top, 15px);
  line-height: 1;
  padding-bottom: 0;

  .tight {
    padding: 0;
  }

  a {
    color: $highlight;
    text-decoration: none;

    &:hover {
      text-decoration: underline;
    }
  }

  span {
    display: inline-block;
  }
}

small {
  color: $navy;
}

Styles are defined mobile-first, with media queries added to adjust for each larger device width/height.

.box {
  margin: 0 auto;
  width: 94%;

  @media screen and (min-width: 769px) {
    width: 700px;
  }
}

Use the breakpoint mixin to reduce media query boilerplate.

.box {
  margin: 0 auto;
  width: 94%;

  @include breakpoint(desktop) {
    width: 700px;
  }
}

Use the rem mixin to convert pixel values to rem with pixel fallback.

.box {
  margin: 0 auto;
  width: 94%;

  @include breakpoint(desktop) {
    @include rem(width, 700px);
  }
}

For naming, use semantic elements contained within an anonymously classed container, with additional variants defined by anonymous class.

$highlight: #69e;
$red: #d33;

.box {
  a {
    color: darken($highlight, 10%);
    text-decoration: none;

    &:hover {
      text-decoration: underline;
    }

    &.error {
      color: $red;
    }
  }
}
